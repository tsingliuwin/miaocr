# miaocr 🎯

`miaocr` 是一个基于 Rust 开发的**实时屏幕选区 OCR 工具**。它采用轻量无边框的置顶悬浮胶囊设计，支持用户自由选择屏幕区域进行实时、高频、连续的文字识别，并将结果保存至本地日志与剪贴板中。

本工具专为高效率跨平台办公和开发调试设计，目前在 Windows 与 macOS 上提供了极好的原生支持，并且在本地集成了多种顶级的离线与云端 OCR 引擎。

---

## 🌟 核心特性

- 📌 **置顶悬浮卡片**：无边框、半透明、支持拖拽的悬浮窗，随时折叠/展开，不占用宝贵的屏幕空间。
- 🔍 **动态屏幕选区**：点击“选区”按钮隐藏自身，支持跨多显示器拖拽选择要识别的任意矩形区域。
- ⚡ **智能预处理**：
  - 对小于 500x400 的小区域选区自动进行 **2x Catmull-Rom 插值放大与锐化**，大幅提高极小文字的识别精度。
  - 针对 Windows 原生 OCR 和 Tesseract 自动应用**大津法（Otsu）二值化与智能反色**，完美适配深色/浅色背景。
- 🔄 **自适应轮询**：常驻后台识别线程，根据最近几次识别的实际耗时动态调整下次识别的休眠时间（50ms~1000ms），兼顾高响应与低 CPU 占用。
- 📝 **智能日志归档**：识别结果将自动写入当前目录下的 `ocr_log.txt`。支持结果去重，若选区内容未变，会自动更新对应内容的时间戳区间（格式为 `[开始时间 ~ 结束时间]`），避免刷屏。
- 📦 **开箱即用，零门槛部署**：本地引擎（Tesseract、PaddleOCR、RapidOCR）支持在界面中一键“下载安装”。
- 🌐 **国内网络加速**：所有下载通道集成了多镜像源自动轮询（包括国内代理 Gitee / ModelScope / ghproxy 等），且内置 SSL 证书绕过（`-k`），彻底告别 GitHub 连接失败导致的报错。

## 🛠️ 集成的五大 OCR 引擎

| 引擎名称 | 适用平台 | 依赖与说明 | 特点 |
| :--- | :--- | :--- | :--- |
| **Windows 原生 OCR** | Windows | 无需下载任何依赖，直接调用 WinRT 系统的 OCR 模块。 | 极速、开箱即用、精度高。 |
| **macOS 原生 OCR** | macOS | 无需下载，首次运行时将通过 Swift 在本地编译并调用系统的 Vision OCR。 | 极速、效果出众、系统原生。 |
| **PaddleOCR (本地)** | 跨平台 | 本地下载 `PaddleOCR-json` 引擎包。**CPU 必须支持 AVX 指令集**。 | 识别极其精准， mkldnn 加速，速度快。 |
| **RapidOCR (本地)** | 跨平台 | 本地下载 `RapidOCR-json` 引擎包。基于 ONNX Runtime，**无 AVX 指令集限制**。 | 内存占用低（~500MB），适合老旧或低配电脑。 |
| **Tesseract (本地)** | 跨平台 | 一键安装系统 Tesseract 程序并下载 `chi_sim` 简体中文语言包。 | 老牌离线开源引擎。 |
| **自定义 API** | 跨平台 | 支持将截取的图像通过 HTTP POST 请求发送至用户自建的云端 API。 | 灵活，可接入各种云端 OCR 服务。 |

## 🚀 快速开始

### 运行环境要求

- **Windows**：Windows 7 x64 及以上。
- **macOS**：macOS High Sierra 10.13 及以上（支持 M 系列和 Intel 芯片）。

### 编译与运行

由于本项目使用 Rust 编写，首先请确保您已安装 [Rust 工具链](https://rustup.rs/)。

1. 克隆或解压项目源码：
   ```bash
   cd miaocr
   ```
2. 在发布模式（Release）下编译并运行：
   ```bash
   cargo run --release
   ```
3. 编译完成后，您可以在以下目录找到独立的可执行文件：
   - Windows: `target/release/miaocr.exe`
   - macOS: `target/release/miaocr`

## 📖 使用指南

1. **拖拽移动**：鼠标长按左上角的 `miaocr` 标题，即可自由拖动悬浮窗位置。
2. **选择区域**：
   - 点击 **“选区”** 按钮，整个屏幕会变成灰色半透明遮罩。
   - 按住鼠标左键并拖拽，框选需要进行 OCR 的目标区域。
   - 松开左键，悬浮窗将还原，并立刻启动该选区的**持续自动识别**。
3. **播放与暂停**：选区生效后，标题栏右侧会出现“暂停”/“继续”按钮，点击可控制后台的轮询识别。
4. **切换引擎**：
   - 展开折叠面板，在“引擎”下拉菜单中选择您想使用的后端。
   - 若选择的本地引擎显示红叉 `✗`，直接点击右侧的 **“安装”**（或 **“重试”**）按钮，工具将自动从最优节点下载、解压并注册该引擎。
5. **获取结果**：
   - 识别文字将直接渲染在多行文本框中，你可以直接编辑或点击右上角的 **“复制”** 复制到剪贴板。
   - 识别结果将同步以时间区间形式增量写入同一目录下的 `ocr_log.txt` 中。

---

## 🗂️ 目录与文件说明

* **运行日志**：位于本地应用数据目录：
  - Windows: `C:\Users\<您的用户名>\.miaocr\miaocr.log`
  - macOS: `~/.miaocr/miaocr.log`
* **引擎缓存**：
  - 自动下载的本地 OCR 可执行程序与模型存放在 `~/.miaocr/PaddleOCR-json/` 和 `~/.miaocr/RapidOCR-json/`。
  - Tesseract 语言包位于 `~/.miaocr/tessdata/`。
* **识别记录**：当前目录下的 `ocr_log.txt`。
